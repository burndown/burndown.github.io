OpenWrt X64 编译

Posted on 2022-11-09

本文介绍两种OpenWrt固件编译的方式，分别是本地 Ubuntu VM 编译和 Github Action 编译。

本文以 Lean 和 Kenzok8 的库为基础进行编译，编译过程选择了一些自己常用软件包，并加了一些自己的配置文件。本地编译流程按照 Lean 的说明，Github Action 编译按照 P3TERX 的说明。

两种方式没有本质的区别，区别在于，一个是本地编译，需要一台 Ubuntu 电脑或者 VM 和良好的网络。一个是在 Github 编译，需要 Github 账号。

准备

OpenWrt 库

下边三个库里包含了我们需要的软件包。Lean 的库提供里 OpenWrt 的源码，Kenzok8 和 Small 提供了很多额外的软件，比如 DDNS, Passwall 等。

Lean: https://github.com/coolsnowwolf/lede

Kenzok8: https://github.com/kenzok8/openwrt-packages

small: https://github.com/kenzok8/small

编译

本地Ubuntu 模式

跟着 Lean OpenWrt 库里的说明操作就好。

注意事项：

  • 如果后续还有可能编译，记得给 Ubuntu VM 划分的磁盘大点，不然编译几次就没空间了
  • 准备好科学环境，不然从 Github 上拉取源码会比较费劲
  • 拉取 Lean 的 代码后，记得在 feed.conf.default 里添加 Kenzo 和 Small 的库
  • 选择镜像的格式，根据自己的情况选择: leagcy or EFI
  • Luci 有默认选项，包含了很多APP，可以根据个人喜欢勾掉，自己手动选择
  • ssrplus 可能和 Passwall 有冲突，选择一个即可
  • 可以在lede 文件夹下创建 files，把自己想预加载的配置文件放进去，但是我这次没有尝试

漫长的等待后，就可以去 ~/lede/bin/targets/x86/64 下看到编译好的固件了。

Github 编译模式

编译过程按照 P3TERX 的说明

简明步骤如下：

  1. 以 P3TERX Action-OpenWrt 为模板创建一个自己的 Action。

  2. 把自己本地生成的 .config 文件上传到根目录里

  3. 修改 diy-part1.sh 把 kenzo 和 small 库加进去

  4. 去 Action 里 执行 Build OpenWrt

  5. 不用执行 update-checker

    漫长的等待后，就可以去 Action 页面下载编译好的固件了。

总结

Ubuntu VM 编译和 Github Action 库编译没有本质的区别，需要的准备工作都是 OpenWrt 和 luci 包的源码，自己需要包含的app列表，预加载的文件等。

不过 Github 有良好的网络环境，这是国内不具备的，所以优势还是很明显的。

Ultra Mobile Pay as You Go 在中国激活 WIFI Calling

Posted on 2022-11-07 Edited on 2022-11-09

本文简单介绍了在国内激活 Ultra Mobile Pay as You Go 电话卡 WIFI Calling 的网络设置。

背景介绍

淘宝入手一张 Ultra Mobile Pay as You Go 手机卡，主要用来注册国外服务，接收短信。月租3美元，包含100分钟通话和100条短信。可以在大陆漫游，漫游期间无法通过运营商网络使用套餐的电话和短信，但是可以通过 WIFI Calling 使用套餐内的电话和短信。

WIFI Calling 是一项电信增值服务，手机可以通过家用或者公共 WIFI 接打电话或收发短信，是对运营商网络覆盖薄弱地区的补充。WIFI Calling 需要手机和运营商同时支持才能使用。Ultra Mobile支持这项服务，所以只要你手机支持 WIFI Calling，在运营商侧和手机侧激活了 WIFI Calling 后，就可以使用这张电话卡，在大陆地区通过 WIFI Calling 网络接收短信。

使用这张电话卡，激活 WIFI Calling 是关键，没有激活 WIFI Calling 的手机卡是无法接收短信的。激活 WIFI Calling 需要手机通过 WIFI 去访问 T-Mobile 的 WIFI Calling 服务器，但是这个服务器的因为各种原因，在国内经常无法访问。淘宝上卖这种电话卡的商家基本都提供了代激活功能，但是你也可以自己动手操作。

激活 WIFI Calling 的方法

以下办法假定你已经激活运营商侧和手机侧的网络，只是无法连接到 WIFI Calling 服务器。请参考 IOS 手册去激活手机侧的 WIFI Calling， 请登陆运营商网站或者联系淘宝卖家获取激活运营商侧的 WIFI Calling 指南。

激活 WIFI Calling 的关键是手机能通过 WIFI 访问到 WIFI Calling 服务器。这里包含两步，第一，DNS 能够正常解析到 WIFI Calling 服务器的 (下文简称服务器) IP 地址，第二，手机能通过 WIFI 访问到服务器的 IP 地址，建立连接。通过测试发现，服务器 IP 地址在国内能够正常访问，但是 DNS 不能正常解析服务器的域名。所以解决办法就是使用一个可以正常解析服务器域名的 DNS。

激活 WIFI Calling 的两种办法：

  • 手机上手动设置 DNS：手动设置一个好用的 DNS
  • 路由器上设置域名劫持：在路由器上直接添加服务器的域名和 IP 地址

如果路由器上有代理
